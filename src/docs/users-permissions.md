# Usuários e Permissões

## O que é esta tela?

Painel administrativo para gestão de usuários, controle de acesso por papel e monitoramento de atividade e uso do sistema.

## Perfis de acesso

| Papel | Permissões |
|---|---|
| **Admin** | Acesso completo ao tenant — todos os módulos, incluindo esta tela |
| **Usuário** | Acesso restrito; o admin define quais módulos são visíveis |

## Status do usuário

| Status | Significado |
|---|---|
| **Aprovado** | Acesso liberado ao sistema |
| **Pendente** | Cadastro aguardando aprovação do admin |
| **Rejeitado** | Acesso negado pelo admin |

## Gestão de usuários

### Aprovar ou rejeitar um usuário
Selecione o usuário na lista e altere o status para **Aprovado** ou **Rejeitado**. O usuário receberá acesso imediatamente após a aprovação.

### Alterar papel
Mude o papel entre **Admin** e **Usuário** conforme a responsabilidade da pessoa.

### Mover para outro tenant
Administradores da plataforma podem transferir um usuário para outro tenant através do painel de edição.

## Monitoramento de sessões ativas

O painel **Usuários online** exibe em tempo real:
- Usuários com sessão ativa no sistema
- Endereço IP e navegador
- Última atividade registrada

## Uso do Assistente IA

O painel **Uso do Assistente IA** mostra por usuário:
- Número de consultas realizadas
- Total de tokens consumidos
- Custo estimado em USD

## Visão por tenant *(admin da plataforma)*

Administradores da plataforma têm acesso à visão consolidada de gastos com o assistente IA por tenant, com totais de queries, tokens e custo.

## Gestão do tenant

O admin pode editar o nome do tenant diretamente nesta tela.

## Dicas

> Revise regularmente os usuários pendentes — novos cadastros ficam em espera até aprovação manual.

> Mantenha o número de admins limitado para garantir controle sobre alterações críticas.

> Use o monitoramento de sessões para identificar acessos suspeitos ou inatividade de usuários.
